# 運転中に使える英会話アプリ構想（1年・平日200〜220日）

## 1. コンセプト

- **対象**: 英語ほぼ初心者。
- **ゴール**: 平日朝20〜30分を1年間継続し、日常〜旅行・仕事の基本会話を自走できる状態。
- **利用シーン**: 運転中のハンズフリー。画面注視不要、音声のみで完結。
- **体験方針**: 「短い往復会話」「即時フィードバック」「復習の自動最適化」。

## 2. 1回20〜30分レッスンの型

1. **ウォームアップ（3分）**
   - 前日までの頻出フレーズを3〜5個、音声でシャドーイング。
2. **新規インプット（7分）**
   - 当日のテーマ（例: 自己紹介、道案内、買い物）を2〜3表現。
3. **対話ドリル（10〜15分）**
   - AIチューターとロールプレイ。
   - 1ターンは短文中心（5〜10語）で成功体験を積む。
4. **要約と宿題（3〜5分）**
   - 「今日言えたこと」「明日1回だけ言う文」を音声で確認。

## 3. 年間カリキュラム（平日200〜220日）

### フェーズA（Day 1〜40）音声慣れ・基礎文
- ゴール: 反射で出る短文を50〜80個。
- 内容: be動詞、一般動詞、疑問文、依頼表現（Can you...?）。

### フェーズB（Day 41〜90）生活会話
- ゴール: 1往復2〜3ターンの会話。
- 内容: 朝のルーティン、家族、天気、移動、買い物、レストラン。

### フェーズC（Day 91〜140）実用会話
- ゴール: 困りごと・確認・言い換えができる。
- 内容: 病院、ホテル、道を尋ねる、電話、予約変更。

### フェーズD（Day 141〜180）仕事・雑談
- ゴール: 自分の仕事説明、簡単な報告・相談。
- 内容: 職種説明、予定共有、進捗、依頼、謝罪、提案。

### フェーズE（Day 181〜220）統合運用
- ゴール: 初見トピックでも3〜5分会話継続。
- 内容: フリートーク、弱点補強、試験形式ロールプレイ。

## 4. 運転中UXの必須要件

- **完全音声操作**: 「次へ」「もう一回」「ゆっくり」「日本語で」などのコマンド。
- **低遅延応答**: 往復1秒台を目標（体感テンポ最優先）。
- **割り込み耐性**: ナビ音声や信号停止で中断→「続きから」で再開。
- **安全優先設計**: 画面タップ不要、長文表示依存なし。
- **オフライン救済**: 通信不安定時は事前DL済みドリルへ自動切替。

## 5. AIチューター設計

- **役割固定**: 優しいコーチ人格（励ます、詰め込まない）。
- **訂正方針**: 会話を止めず、ターン後に1点だけ修正。
- **難易度制御**: 成功率70〜85%を保つよう語彙・速度を調整。
- **言い換え支援**: 「別の言い方を2つ」を必ず提示。
- **日本語ブリッジ**: 詰まったら日本語ヒント→英語再挑戦。

## 6. 学習ロジック（継続最適化）

- **SRS復習**: 間違えた表現を1日後・3日後・7日後に再出題。
- **朝特化スケジューラ**: 平日朝のみ通知、未実施は昼に自動リカバリ。
- **週間レビュー**: 金曜に5日分を圧縮復習（10分）。
- **月次チェック**: CEFR A1→A2相当の到達判定テスト。

## 7. MVP（最初の8週間で作る範囲）

1. 音声入出力（STT/TTS）
2. 50レッスン分の台本と自動ドリル
3. 進捗管理（連続日数、習得フレーズ数）
4. 誤答ログとSRS復習キュー
5. 車載向けの「片手不要UI」

## 8. 技術スタック案

- **モバイル**: React Native（iOS/Android同時開発）
- **音声認識**: ストリーミングSTT（低遅延重視）
- **音声合成**: 自然な英語TTS（速度可変）
- **対話制御**: LLM + プロンプトテンプレート + 進捗状態管理
- **データ**: SQLite（端末） + クラウドDB（同期）
- **分析**: 発話成功率、応答遅延、継続率、再学習率

## 9. KPI（成功指標）

- 4週継続率: 60%以上
- 12週継続率: 40%以上
- 1セッション平均往復ターン: 12以上
- 発話後のAI応答中央値: 1.5秒以下
- 6か月時点でのA2到達率: 50%以上

## 10. 次アクション（実行順）

1. 主要ペルソナを2種類に絞る（完全初心者/中学英語やり直し）。
2. Day1〜20の台本を先に作る（毎日20分版）。
3. 運転中コマンド語彙を先行実装（次へ・繰り返し・日本語ヒント）。
4. 10人で2週間の実地テスト（朝の通勤時）。
5. 継続率と遅延を見て、教材量よりテンポを優先して改善。
